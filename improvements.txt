Improvements / Follow-up Tasks

- allow hosts to remove users (later can give them the option to add a reason as to why they're removing that user)

1. Button outline spinner color (Button.tsx:46)
   The outline variant's ActivityIndicator spinner is hardcoded to colors.primary (teal).
   On dark backgrounds (e.g. welcome screen), this would be invisible if the button ever
   enters a loading state. Update Button.tsx to accept a custom spinner color or infer it
   from the background context.

2. Accessibility labels on Sign Up / Log In buttons (welcome.tsx)
   The Button component does not forward an accessibilityLabel prop. The "Sign Up" and
   "Log In" buttons on the welcome screen have no explicit accessibility label. Update
   the Button component to accept and forward an accessibilityLabel prop, then add labels
   to both buttons on the welcome screen.

3. Duplicate logo in login.tsx and signup.tsx
   Both login.tsx and signup.tsx render their own "LetsPlay / Find your next game" header.
   Now that the welcome screen owns the branded entry point, users navigating
   Welcome → Log In or Welcome → Sign Up see the logo twice. Remove the logo/header
   section from login.tsx and signup.tsx in a follow-up PR.

4. Runtime import of Ionicons in sports utility (lib/utils/sports.ts:1)
   The file imports Ionicons at runtime but only uses it for the return type annotation
   of getSportIcon. Change to `import type { Ionicons }` to avoid pulling the icon
   library into a pure utility module's bundle.

5. useLocalSearchParams id type safety (app/event/[id].tsx:26)
   Expo Router's useLocalSearchParams types `id` as `string | string[]`. Both
   useEventDetail and useJoinEvent accept `string`. Normalize with:
   `const eventId = Array.isArray(id) ? id[0] : id ?? '';`
   Low risk in practice since [id].tsx always receives a string, but would satisfy
   strict TypeScript.

6. "How did you hear about us?" referral question (app/(auth)/onboarding-profile.tsx)
   Add an optional referral source selector at the bottom of Step 3 (onboarding-profile),
   below the About Me field. Options: Social media, Friend/word of mouth, Google search,
   App store, Other. Store as `referral_source` column on the profiles table (nullable text).
   Purpose: internal analytics only — never shown in the UI after collection.

7. useJoinEvent has no guard on empty eventId (lib/hooks/useJoinEvent.ts)
   Unlike useEventDetail which has `enabled: !!eventId`, the mutation has no protection
   against being called with an empty string. Add a guard at the top of mutationFn:
   `if (!eventId) throw new Error('No event ID provided.');`
